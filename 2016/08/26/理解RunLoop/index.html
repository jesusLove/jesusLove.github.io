理解RunLoop

繁华落尽丶千2016-08-262023-11-02

好记性不如烂笔头。本文是根据官方文档整理的笔记，理论性比较强，基础知识，如有错误还请指正。

Run Loops是线程相关的基础框架的一部分。runloop就是一个事件处理的循环，用来不停的调度工作及处理输入事件。使用runloop的目的是让你的线程在有工作的时候工作，没工作的时候休眠。

RunLoop解析

Run loop 本身听起来就和像它的名字。它是一个循环，你的线程进入并使用它来运行响应输入事件的事件处理程序。

Run loop接收事件来自两种不同来源：输入源（input source）和定时源（timer source）。

  • 输入源传递异步事件，通常消息来自于其他线程或程序。
  • 定时源则传递同步事件,发生在特定的时间或者重复的时间间隔。

两种源都使用程序的某一特定的处理例程来处理到达的事件。

下图显示Runloop的概念结构以及各种源。

由图：

  • 输入源 传递一步消息给相应的处理线程，并调用runUntilDate：方法退出。
  • 定时源 直接传递消息给处理线程，但是并不退出run loop。

另外，run loops也会生成关于run loop行为的通知。添加Observer用来监测runloop的运行状态。

RunLoop模式

RunLoop模式是一个集合，包含所有要监视的输入源和定时源以及要通知的runloop注册观察者。当每次运行Runloop时，需要指定运行模式，只有和当前模式相关的源才会被监视并允许传递消息。

苹果公开提供的Mode有两个:

  • NSDefaultRunLoopMode 默认的状态
  • UITrackingRunLoopMode UIScrollView滑动状态

另外，提供了一个操作Common标记的字符串：NSRunLoopCommonModes

Input Source（输入源）

输入源异步的发送消息给线程。输入源分为两种：

  • 基于端口的输入源：监听程序响应的端口。
  • 自定义的输入源：监听自定义的事件源。

RunLoop并不关心是哪种输入源。系统会实现两种输入源供使用。两种输入源的区别：**基于端口的输入源由内核自动发送，而自定义的输入源需要人工从其他线程发送。

基于端口的输入源

Cocoa和Core Foundation内置支持使用端口相关的对象和函数创建基于端口的输入源。例如：

  • 在Cocoa中，不用直接创建输入源。只需简单创建端口对象使用NSPort的方法把端口添加到runloop中。端口对象会自动创建和配置输入源。
  • 在Core Foundation中，必须人工创建端口和Runloop源，在两种情况下，可以使用端口相关的函数（CFMachPortRef, CFMessagePortRef, CFSocketRef）来创建合适的对象。参考Configuring a Port-Based Input Source
Cocoa执行Selector的源

除了基于端口的源，Cocoa定义了自定义输入源，允许你在任何线程执行selector。参考：NSObject Class Reference

定时源

定时源在预设的时间点同步传递消息。定时器是线程通知自己做某事的一种方法。需要注意的是：定时器可以产生基于时间的通知，但是它并不是实时机制。也就是说，在某种情况下，定时器实际时间和预设时间有误差。

Run Loop观察者

runloop观察者是在runloop特定状态触发响应。

  • RunLoop 入口
  • RunLoop 何时处理一个定时器
  • RunLoop 何时处理一个输入源
  • RunLoop 何时进入睡眠状态
  • RunLoop 何时被唤醒，但在唤醒之前要处理的时间
  • RunLoop 终止

何时使用RunLoop

仅当为程序创建辅助线程的时候，才需要显示运行一个RunLoop（例如：AFNetWorking）。RunLoop是程序主线程的关键部分，在程序启动的时候会自动启动RunLoop。对于辅助线程，需要判断RunLoop是否必须配置启动，一般如下情况会启动：

  • 使用端口或者自定义输入源来和其他线程通信
  • 使用线程的定时器
  • Cocoa中使用任何performSelector...的方法
  • 使线程周期工作

关于如何配置和退出一个RunLoop，参阅下一节”使用RunLoop对象”的介绍。

使用RunLoop对象

RunLoop对象提供了添加输入源，定时器和RunLoop的观察者以及启动RunLoop的接口。每个线程都有唯一的与之关联的RunLoop对象。在Cocoa中，该对象是NSRunLoop类的一个实例。

获取RunLoop对象

为了获得当前线程的RunLoop，你可以采用以下任一方式：

  • 在Cocoa程序中， 使用NSRunLoop的currentRunLoop类方法来检索一个NSRunLoop对象。
  • 使用CFRunLoopGetCurrent函数。

虽然并不是相同的类型，当时可以从NSRunLoop对象中获取CFRunLoopRef类型。NSRunLoop类定义了一个getCFRunLoop方法，该方法返回一个CFRunLoopRef类型。

配置RunLoop

在辅助线程运行RunLoop之前，必须添加一个输入源或定时器给它。如果RunLoop没有任何需要监视的话，会在创建完立马退出。如何添加源，参阅”配置RunLoop源“。

另外，我们可以添加RunLoop观察者来监视RunLoop的不同执行阶段情况。为了给RunLoop添加观察者，可以创建CFRunLoopObserverRef不透明类型，并使用CFRunLoopAddObserver将它添加到RunLoop。RunLoop的观察者必须有Core Foundation函数创建。

显示创建一个RunLoop观察者

启动RunLoop

启动RunLoop只对程序的辅助线程有意义。一个RunLoop通常必须包含一个输入源或定时器监控时间。如果没有，RunLoop启动后立即退出。

退出RunLoop

两种方法可以让RunLoop处理时间前退出：

  • 给RunLoop设置超时时间
  • 通知RunLoop停止

使用CFRunLoopStop来显式的停止RunLoop和使用超时时间产生的结果相似。RunLoop把所有的剩余的通知发送出去再退出。

线程安全和RunLoop对象

CoreFoundation中的函数通常是线程安全的，可以被任意线程调用。但是如果修改了RunLoop的配置然后需要执行这些操作，任何时候你最好还是在RunLoop所属的线程中执行这些操作。Cocoa的NSRunLoop类不是线程安全的，修改RunLoop，你必须RunLoop所属的线程中里面完成这些操作。
